Department of Homeland Security
homepage and navigation redesign.

Role: UX/UI Designer
Software: Figma, Miro, Invision, Google Apps

OVERVIEW

Website Usability Testing

We conducted

6

usability tests of the current website

Key Takeaways:

  • Users all went to the search bar to find information directly rather than browsing the site
  • Users quickly scanned pages to find keywords but were unable until to find information until they slowed down to read the page.
  • Based on current navigation options, users were unsure how to start browsing the website to complete assigned tasks.

“When going to a website I have a objective and narrow focus, that is why I use the search.”

Navigation Usability Testing

We conducted

4

usability tests of the current website

Key Takeaways:

  • Navigation categories are broad
  • Users were not sure what was under each category
  • Had a lot of layers to go through to find what they wanted

“ It would be nice to have a dropdown to see what’s within each option.”

PROBLEM STATEMENT

THE PROBLEM
THE SOLUTION
The current DHS website has a lot of information but it is hard to navigate without going straight to the search bar. It leaves users confused on where to start and doesn’t give a clear message of who they are.
The new website would benefit from having their main information and purpose displayed on the Homepage, easily accessible for users.
Changing the way users navigate the website by having a new navigation will allow users to see all their options at a quick glance.

DESIGN PROCESS

LoFi Wireframe

LoFi wireframing of the homepage and navigation changes. Based on user testing a hover/drop down menu was implemented

HiFi Wireframe

New hover state allows users to see what is available under each section without clicking to another page.
This allows users to find information easier without going directly to the search.
